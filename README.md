Below are examples of common ticket workflows that demonstrate how issues move through the osTicket system from creation to resolution.

### **1. Password Reset Request**

2. Ticket automatically assigned to **IT Support**.  
3. Agent verifies user identity and resets password.  
4. User confirms successful login.  
5. Ticket closed and marked as resolved.

---

### **2. Laptop Not Booting**
1. Employee submits a hardware issue ticket.  
2. Ticket routed to **Desktop/Hardware Support**.  
3. Agent performs diagnostics (power checks, hardware tests).  
4. Issue repaired or device replaced.  
5. Agent documents actions taken and closes the ticket.

---

### **3. Software Installation Request**
1. User submits request for software (e.g., Adobe Acrobat).  
2. Ticket assigned to **Software Team**.  
3. Agent verifies license availability and approval.  
4. Software installed and user notified.  
5. Ticket closed.

---

### **4. Network Connectivity Issue**
1. User reports no internet or slow connection.  
2. Ticket routed to **Networking Department**.  
3. Agent checks switch port, runs ping tests, resets connection.  
4. Issue resolved (e.g., cable replacement, DHCP refresh).  
5. Ticket closed after user confirmation.

---

### **5. New Employee Onboarding**
1. HR submits new hire setup request.  
2. Ticket assigned to multiple teams (IT, Security, Facilities).  
3. Each team completes their assigned tasks (accounts, badges, workspace).  
4. All updates recorded in ticket.  
5. Ticket closed once onboarding is fully completed.
